BASES DE DATOS ORIENTADAS A COLUMNAS

CASSANDRA

Cassandra almacena los datos como almacenes clave-valor. Permite definir tablas mediante filas y columnas, pero la estructura tabular no se utiliza en el almacenamiento real. En su lugar, utiliza el modelo de base de datos de columnas anchas, por lo que cada fila de la tabla puede tener un conjunto de columnas diferente. Puede agrupar las columnas en familias de columnas según su tipo de datos o su uso. Cada fila tiene una clave principal que puede utilizar para leer rápidamente los datos de Cassandra.

Características:
  1. Escalabilidad horizontal
    Cassandra está diseñada para crecer fácilmente a través de múltiples servidores, ofreciendo una alta tolerancia a fallos. Si un servidor falla, los datos permanecen disponibles en otros nodos de la red.
  2. Alto rendimiento
    Prioriza la lectura y escritura rápida de grandes volúmenes de datos, haciéndola idónea para aplicaciones que manejan información constantemente cambiante.
  3. Disponibilidad siempre activa
    Cassandra está pensada para funcionar sin interrupciones, evitando la pérdida de datos.
  4. Modelo de datos flexible
    Utiliza un modelo de datos flexible que permite almacenar información en columnas, facilitando la adaptación a estructuras cambiantes.
