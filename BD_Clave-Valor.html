BASES DE DATOS CLAVE - VALOR

¿Qué son las Bases de Datos Clave - Valor?

Las bases de datos clave-valor (BDCV), también conocidas como almacenes clave-valor, son un tipo de noSQL que almacena datos como un conjunto de pares clave-valor. En una BDCV, cada dato se almacena como un par: una clave única que identifica al dato y un valor que contiene los datos en sí. Las claves suelen ser cadenas de caracteres, pero también pueden ser números u otros tipos de datos. Los valores pueden ser de cualquier tipo, incluyendo cadenas de caracteres, números, imágenes, objetos binarios, etc.

REDIS

Redis es un almacén de datos clave-valor en memoria de código abierto, muy popular y utilizado con frecuencia. Funciona como una especie de diccionario gigante que permite almacenar y recuperar datos rápidamente. A diferencia de las bases de datos tradicionales que guardan información en un disco duro, Redis almacena todo en la memoria RAM de la computadora, lo que lo hace extremadamente veloz para acceder a la información.

Funcionamiento:

A diferencia de las bases de datos relacionales tradicionales, Redis no almacena datos en un disco duro, sino en la memoria principal del servidor. Esto permite que Redis sea extremadamente rápida para leer y escribir datos. Redis también utiliza una arquitectura distribuida, lo que significa que se puede escalar horizontalmente agregando más servidores al clúster.

Características:
  1. Velocidad
    Al guardar los datos en memoria RAM, la lectura y escritura de información se realiza a velocidades vertiginosas, superando con creces a las bases de datos tradicionales.
  2. Flexibilidad
    Ofrece soporte para diversas estructuras de datos, incluyendo cadenas de texto, listas, conjuntos ordenados, conjuntos desordenados y hashes. Esta versatilidad permite almacenar diferentes tipos de información de manera eficiente.
  3. Funcionalidad adicional
    Proporciona características adicionales como persistencia opcional en disco para respaldo, pub/sub para mensajería entre aplicaciones y contadores atómicos para manejar valores que cambian constantemente.
  4. Aplicaciones
    Es ideal para tareas como el almacenamiento en caché, la administración de sesiones de usuario en aplicaciones web, clasificaciones (rankings), publicación/suscripción de mensajes en tiempo real y colas de trabajo.
Desventajas:
  1. Falta de estructura
    Redis no tiene una estructura de datos definida, lo que puede dificultar la consulta de datos complejos. Esto puede requerir un mayor esfuerzo en la gestión de datos y en la creación de consultas para recuperar información específica.
  2. Almacenamiento en memoria
    Redis almacena datos en memoria, lo que significa que los datos se pueden perder si el servidor falla. La cantidad de datos que se pueden almacenar en memoria está limitada por la capacidad de la RAM del servidor. Escalar horizontalmente implica agregar más servidores al clúster, lo que aumenta la complejidad de la gestión.
  3. Soporte de consultas limitado
    Redis no dispone de un lenguaje de consulta estructurado como SQL, lo que limita la forma en que se pueden recuperar y manipular los datos.
